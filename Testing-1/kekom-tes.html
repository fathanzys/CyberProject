Hybrid Cryptography File Sharing

Top-notch file security with the powerful combination of RSA and AES, a fast, secure, and trusted hybrid technology!

About

Dalam proyek ini, kami mengimplementasikan sistem kriptografi hybrid yang menggunakan AES (Advanced Encryption Standard) untuk enkripsi file dan RSA (Rivest-Shamir-Adleman) untuk mengenkripsi kunci AES. Tujuan utama dari proyek ini adalah untuk memastikan transmisi file yang aman dengan cara mengenkripsi file tersebut dan hanya memungkinkan penerima yang sah untuk mendekripsinya. Sistem ini akan mendukung berbagai ukuran file (1MB, 5MB, 10MB, dan 100MB) dan bertujuan untuk mengevaluasi kinerja dari segi waktu enkripsi/dekripsi dan konsumsi sumber daya (RAM/CPU).

Proyek ini juga mencakup antarmuka pengguna yang ramah, memungkinkan pengguna untuk mengunggah file, memilih algoritma enkripsi, dan menyimpan file yang sudah terenkripsi. Setelah proses enkripsi, file tersebut dapat dikirim melalui protokol pengiriman yang disimulasikan, dan dekripsi dapat dilakukan oleh penerima yang dimaksud. Sistem secara keseluruhan menjamin kerahasiaan, integritas, dan keaslian file yang ditransfer.

Selain fungsi dasar, analisis kinerja akan dilakukan untuk menilai efektivitas sistem kriptografi hybrid ini, dengan fokus pada kecepatan dan penggunaan sumber daya. Simulasi serangan brute-force juga akan dilakukan untuk menguji keamanan proses enkripsi dan menilai ketahanannya terhadap jenis-jenis serangan siber yang umum.

AES (Advanced Encryption Standard)

AES adalah algoritma kriptografi simetris yang digunakan untuk mengenkripsi dan mendekripsi data menggunakan kunci yang sama. AES bekerja dalam blok 128 bit dan dapat menggunakan kunci 128, 192, atau 256 bit. Proses enkripsi AES mencakup serangkaian transformasi matematis pada data, seperti substitusi byte, pergeseran baris, dan pencampuran kolom.

AES digunakan dalam banyak aplikasi untuk mengamankan data dalam jumlah besar dan dikenal karena kecepatan dan efisiensinya. AES 256-bit dianggap sangat aman dan banyak digunakan dalam aplikasi enkripsi file, VPN, dan komunikasi aman.

Kelebihan: Cepat, efisien, dan aman (terutama AES 256-bit).
Kelemahan: Memerlukan cara aman untuk distribusi kunci simetris.

RSA (Rivest–Shamir–Adleman)

RSA adalah algoritma kriptografi kunci publik yang menggunakan pasangan kunci publik dan pribadi untuk enkripsi dan dekripsi. RSA dianggap aman karena kesulitan dalam faktorisasi angka besar yang menjadi dasar keamanannya. RSA digunakan dalam tanda tangan digital dan enkripsi data sensitif.

Kelebihan: Keamanan didasarkan pada kesulitan faktorisasi dan memungkinkan tanda tangan digital.
Kelemahan: Lebih lambat daripada enkripsi simetris (seperti AES) dan memerlukan kunci yang lebih besar untuk tingkat keamanan tinggi.
